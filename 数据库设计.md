# 校园一卡通系统数据库设计（OpenGauss）

## 一、数据库概述

本系统基于 OpenGauss 数据库设计，实现校园一卡通系统的完整业务功能。

系统按照模块划分进行数据库设计，涵盖：

- 核心业务模块
- 辅助业务模块
- 系统管理模块

数据库采用：

- 逻辑外键（不使用物理外键）
- 软删除机制
- 全流程数据记录

## 二、设计规范

### 2.1 逻辑外键

所有关联关系使用 `xxx_id` 维护，不使用 FOREIGN KEY。

---

### 2.2 软删除

统一字段：

```sql
is_deleted INT DEFAULT 0
````

---

### 2.3 通用字段

```sql
create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
update_time TIMESTAMP,
is_deleted INT DEFAULT 0
```

```sql

    id BIGSERIAL PRIMARY KEY,

    college VARCHAR(100),

    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

    is_deleted INT DEFAULT 0
```

## 七、设计总结

本系统数据库特点：

1. 模块划分清晰（核心 + 辅助 + 系统）
2. 采用逻辑外键，提高扩展性
3. 所有数据支持软删除
4. 关键操作全部留痕（流水 + 日志）
5. 支持后续高并发与大数据扩展
